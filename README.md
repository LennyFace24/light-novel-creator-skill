## 我们的核心理念

长篇小说创作中，AI 面临两个根本矛盾：

**矛盾一：AI 记不住** — 上下文窗口有限，写到后面忘了前面的人设、伏笔、战力设定。

**矛盾二：AI 写不长** — 字数不够时 AI 倾向于"原地堆砌描写"而非"往前推进剧情"，导致篇幅缩水但故事毫无进展。

这个技能就是为了解决这两个矛盾而设计的。

## 如何让 AI 驾驭长文本而不遗忘

### 全量标准化 Markdown 持久化架构

我们不依赖 AI 的上下文记忆，而是将所有关键信息**持久化到磁盘文件**中，每卷创作时重新加载。这就像作者手边的设定本——随时翻阅，不会遗忘。

```
project/
├── global_characters.md          # 全局角色静态属性库
├── global_timeline.md            # 全局时间轴总纲
├── global_foreshadowing.md       # 全局伏笔追踪总表
├── global_story_outline.md       # 全局故事主线总纲
├── Volume_01_卷名/
│   ├── Volume_01_text.md         # 本卷正文
│   ├── Volume_01_characters.md   # 本卷角色卡
│   └── Volume_01_summary.md      # 本卷梗概+字数归档
└── ...
```

### 三大防遗忘机制

**① 就近完整继承规则**
创作新一卷时，每个角色的角色卡必须从最近一卷完整复制全部过往经历，再追加本卷新信息。AI 无需记忆——每次打开角色卡就是角色的完整一生。

```
上一卷角色卡（完整复制）
    ↓
添加本卷新增经历
    ↓
最新角色卡 = 全部过往 + 本卷更新
```

**② 全局文件 → 分卷文件 双重校验**
- `global_characters.md` 保存角色的恒定设定（种族、本源能力等），永久固定
- 各卷 `Volume_XX_characters.md` 保留角色在该卷的动态状态变化，逐卷可追溯
- 人设判定优先级：最新分卷角色卡 > 全局角色静态库

**③ 伏笔追踪总表**
所有伏笔（埋点、回收、废弃）统一登记在 `global_foreshadowing.md`。创作前先读取未回收伏笔清单，确保每条伏笔都有归宿。

## 字数不够时的扩写规则

AI 字数不足时，绝大多数问题出在一个根因：**AI 倾向于把已有场景描写得更细致，而不是增加新的故事事件。**

我们的规则体系围绕"强制故事增量、禁止描写堆砌"展开。

### 第一层：前置预防——容量预估

在创作正文之前，先输出**容量预估表**，将字数目标转化为具体的故事情节数量：

```markdown
【本卷容量预估表】
- 目标总字数：80000 字
- 计划话数：序章 + 12 话正篇 + 尾声
- 所需最小剧情节拍数：36 个（= 12 话 × 3）
- 已规划剧情节拍数：28 个
- 容量缺口：8 个 → 需扩容！
```

**每个剧情节拍 = 一个独立故事事件**（冲突、对话、探索、发现等）。缺口为 0 才允许开始正文创作。这保证了"有足够的故事可写"在动笔前就已解决。

### 第二层：正文创作中的字数感知锚点

每话创作前，大纲中必须嵌入 5 项锚点：

| 锚点 | 含义 |
|------|------|
| 目标字数 | 本话应达到的字数 |
| 核心剧情节点数 | 至少几个独立推进的剧情节拍 |
| 对话场景数 | 至少几段二人以上对话 |
| 描写段落数 | 至少几段独立场景/动作/心理描写 |
| 情绪转折点数 | 至少几次人物情绪变化或立场微调 |

锚点未全部覆盖 = 字数大概率不达标。这提前暴露了"剧情量不够"的问题。

### 第三层：强制续写 + 有效增量检测

每话完成后立即统计字数。若未达标：

```
【字数核验】本话当前 3210 字，目标下限 3500 字，缺口 290 字，启动强制续写。
```

续写必须遵守两条铁律：

**铁律一：只能新增故事事件，禁止描写堆砌**
- ✅ 新增一段角色对话冲突 → 有效增量 100%
- ✅ 增加一段意外遭遇事件 → 有效增量 100%
- ❌ 把"他走了过去"扩写成"他迈着沉重的步伐缓缓走了过去" → 有效增量 0%

**铁律二：有效增量 < 70% 则续写无效，重做**
续写完成后剔除纯环境描写、重复独白、无推进意义的动作描摹，剩下的才是"有效增量字数"。若有效增量占比低于总新增字数的 70%，判定为描写堆砌，整次续写作废重来。

### 第四层：大纲回溯扩容（三种优先级）

如果单纯续写仍无法达标，说明本话大纲的剧情容量本身就不够。此时必须**回溯大纲，增加故事节点**，而非在正文层面硬凑。

| 优先级 | 操作 | 适用场景 |
|--------|------|---------|
| **P1 节点内插入** | 在现有剧情节点之间插入 1-2 个新事件 | 本话结构不受影响，最安全 |
| **P2 展开分支** | 将简要提及的剧情点展开为完整事件 | 有可用素材但写得不够细 |
| **P3 改接口** | 将本话结尾后移，末尾新增事件 + 同步更新下一话开头 | 前两种都不够用时的最后手段 |

### 第五层：串行创作 + 下一话锁定

- **一次只写一话**，严禁先生成全卷初稿再统一扩写
- **本话未达标不得启动下一话**，防止把当前话字数缺口往后累积
- 达标后锁定本话正文，后续不得回头修改

## 量化指标总结

| 规则 | 阈值 | 后果 |
|------|------|------|
| 修饰性描写占新增字数 | > **30%** | 判定无效扩写 |
| 有效增量占新增字数 | < **70%** | 续写无效，重做 |
| 单话字数 | < **3500 字** | 启动强制续写 |
| 单话字数 | < **2000 字** | 禁止出现 |
| 有效剧情字数占总字数 | < **70%** | 校验不通过 |
